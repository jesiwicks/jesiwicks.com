Seedance 2.0

Stop describing.
Start directing.

The Core Concept Mental Model

Seedance 2.0 is reference-first - upload your files, assign each one a role with @ tags, and the model follows your references instead of guessing at them. That's a meaningful difference: 12 files across four input types in a single generation, with an @ reference system that has no direct equivalent in any other model.

The Shift That Changes Everything

Describing guesses.
Directing specifies.

Text prompts leave the decisions to the model - what your character looks like, which camera angle, what "cinematic" means. References take those decisions back. Upload a character and the model holds their face. Upload a clip and it replicates the camera move. Upload audio and it syncs to it. The @ reference system is how you stay in control.

Describing - the model decides

"A woman in a red coat walks through a rainy city street at night, cinematic lighting, dramatic camera move."

The model decides what the woman looks like, which street, which camera move, what "dramatic" means. You get its interpretation - not yours.

Directing - you decide

@Image1 as the character. Scene is the street in @Image2. Replicate camera movement from @Video1. Background music from @Audio1.

You supply the references. The model executes your exact vision - character, scene, camera, and sound - all under your control.

💡 What directing actually does Every @ tag is an instruction the model can't ignore. Tag an image as your character - it locks their face. Tag a clip - it copies the camera move. Tag audio - it syncs to it. Each tag removes one decision from the model and hands it to you.
The @ Tag System

Every file needs
a name and a role.

When you upload files into Omni Reference, each one gets assigned a name. You then use that name in your prompt as an @ tag to tell the model what that file is for. Without the @ tag, the model doesn't know which file to use for what - and results become unpredictable.

1
Upload your files into Omni Reference
Images, video clips, audio, or any combination - up to 12 files total. The platform assigns each file a name (Image1, Video1, Audio1, etc.) automatically when you upload.
2
Trigger the asset picker with @
In the prompt box, type @ anywhere to open the asset picker. Select the file you want to reference. It inserts the tag into your prompt at the cursor position.
3
Tell the model what each file is for
Don't just tag - assign a role. @Image1 as the character, @Video1 for shot movement, @Audio1 for background music. The role is as important as the tag.
4
Describe only what to generate - not what you've already shown
If you've uploaded the character as @Image1, don't describe the character again in text. The reference is your description. Use your prompt words for actions, movement, timing, and style.
⚠ MultiFrames ≠ Omni Reference MultiFrames is a Seedance 1.0 carry-over that appears in the UI. For all Seedance 2.0 multimodal work - images, video, audio, or any combination - use Omni Reference. MultiFrames will not give you 2.0 capabilities.
Why Characters Drift | And How to Stop It

The model prioritizes
motion over identity.

This is the most important technical fact about Seedance 2.0 that most guides don't explain. When the model has to choose between keeping a character's face accurate and making the motion feel alive, it picks motion. Understanding this changes how you build your reference set.

①
Use 2 reference images - not 6
More references introduce more variation for the model to reconcile. Field testing found that cutting references from 6 images to 2 consistent, well-lit images reduced visible character drift by roughly 60%. Less is more.
②
Match the lighting across your reference images
Mixing warm indoor and cool outdoor reference photos causes the model to alternate between skin tones and eye colors across frames. Keep lighting temperature consistent across all character references.
③
Include a character anchor line in every prompt
Add a compact identity block: Character: [name], [distinctive features]. Keep these features unchanged. Then add: No mirrored features. Seedance respects explicit negative anchors more than most models.
④
Slow the motion down to protect the face
Fast movements - especially hand gestures near the face, or quick yaw turns - trigger identity resets. When a hand crosses the face, the model treats the next frame as a fresh reference. Slower, deliberate motion keeps identity stable.

What You Can Upload

Four input types.
One generation.

Seedance 2.0 accepts images, video, audio, and text in a single generation. The combined file limit is 12. Allocate that budget intentionally - each file costs attention from the model.

📷
Images
≤ 9
PNG or JPG. Use for character appearance, scene setting, style reference, first frame, or last frame. Minimum 1080p recommended - higher resolution = better consistency.
🎥
Video clips
≤ 3
Combined max 15 seconds total. Use for shot movement, camera language, action choreography, or as existing footage to extend or edit. Trim clips to the exact moment you need before uploading.
🎵
Audio files
≤ 3
MP3 format, combined max 15 seconds. Use for background music rhythm, atmospheric tone, or beat-sync. You can also reference audio directly from a video file - no separate MP3 needed.
📝

≤ 80w
Plain language up to ~80 words. Describe actions, movement, and timing. Reference every uploaded file with an @ tag. Beyond 80 words the model starts cherry-picking details - shorter is sharper.

The Prompt Formula

Five parts.
Under 80 words.

Every effective Seedance 2.0 prompt follows the same structure, regardless of complexity. The formula scales from a basic text-only generation all the way to a multi-file reference shoot. Learn this and every template below will make immediate sense.

The Universal Prompt Structure
Subject + Action + Scene + Camera + Style
Keep it under 80 words. One camera move per prompt. Be specific about speed: "slowly," "quickly," "gently" - not "dramatically" or "dynamically." If you've uploaded reference files, replace the relevant text description with the @ tag and a role. Don't describe what you've already shown.

Level 3 | Multi-Step Production
Level 3
Chaining clips into a coherent sequence
30–60 minutes · Full workflow · Goal: produce a sequence of clips that hold together visually

Seedance maxes at 15 seconds per generation. For longer content you chain clips - each generation uses the previous output as a reference. The key to making chains work is keeping your character block, color grade, and camera logic identical across every generation.

The 5-Step Production Workflow
1
Build your reference pack before writing a single prompt
Gather: one character headshot (front-facing, clean lighting), one style reference image, and optionally one video clip for camera language. Label them. Trim video refs to the 2–3 seconds you actually need. A strong, disciplined reference pack is the difference between a chain that holds and one that drifts.
2
Write a fixed character block - reuse it in every generation
Create a compact text block containing all immutable character details: Character: [name], [key features - hair, clothing, distinctive marks]. Keep these features unchanged throughout. Copy this block into the top of every single prompt in your sequence. Do not vary it.
3
Generate Shot 1 on Fast - iterate until the look is locked
Run 2–3 Fast generations, changing only one variable per run. Don't change the prompt AND the references at the same time - you won't know what caused the improvement. Once Shot 1 looks right, that output becomes your consistency anchor for everything that follows.
4
Use Shot 1's output as @Video1 for Shot 2
Upload Shot 1's output as a reference into Shot 2's generation. Your prompt: Continue the scene from @Video1. [Character block]. [New action for Shot 2]. Match the color grade and camera height exactly. The previous output carries visual continuity into the next shot.
5
Final renders on Standard - one per shot
Once your sequence is locked at Fast quality, switch to Standard Seedance 2.0 for final renders. Generate one shot at a time. Don't try to regenerate everything at once - if one shot drifts, you only need to redo that one.
⚠ Chain limit - drift compounds after 4–5 clips Each generation introduces small variations. By Shot 5 or 6, even with a consistent reference pack, subtle drift accumulates. For sequences longer than 4–5 clips, rebuild your reference pack from scratch at that point using the latest output as your new anchor.

5 Common Mistakes Hard Limits When to Switch

These are the five mistakes every Seedance 2.0 user makes in the first two weeks. Most of them aren't obvious from the documentation. They come from people burning through credits and documenting what went wrong.

The Five Mistakes

Known failure modes.
Preventable every time.

These are documented from real community experience - not hypothetical edge cases. Every one of them has a specific fix.

01
Forgetting @ tags on uploaded files
You upload a character image and a reference video, write a prompt, and hit generate. The model ignores both files because they were never referenced in the prompt. Without the @ tag, the model doesn't know which file to use for what - it may not use them at all, or mix them up unpredictably.
✓ Fix: Every uploaded file needs an @ tag with a role. Every time. No exceptions. "@Image1 as the character" is the minimum.
02
Too many characters in one scene
Three or more characters in a single generation leads to faces drifting, bodies warping, and someone growing an extra arm. The model's attention is split across too many identity anchors and motion paths simultaneously. It picks motion over identity when attention runs out - and with 3+ characters, it runs out fast.
✓ Fix: Maximum 2 characters per generation. If you need a crowd, make them blurry background elements without identity references. Generate foreground characters separately.
03
Writing a novel instead of a prompt
Prompts over ~100 words cause the model to cherry-pick details at random while ignoring others. The more you add, the less control you have. Creators who are used to Midjourney-style descriptor stacking consistently get worse results in Seedance than those who write tight, structured 40–60 word prompts.
✓ Fix: Subject + Action + Scene + Camera + Style. Under 80 words. If you can't say it in a tweet, cut it. Move detail into your reference images instead.
04
Stacking camera movements
"Pan left while zooming in as the camera tracks the subject" produces a jittery mess that looks like a broken gimbal. The model can execute one camera movement well. It cannot execute three simultaneously. This is not a prompt wording problem - it is a fundamental model constraint.
✓ Fix: One camera move per prompt. Always. A slow dolly-in. A gentle pan. Or a static lock. If you need multiple moves, they go in separate generations.
05
Fast hand gestures and finger movements
"Rapidly gestures while counting on fingers" → extra fingers, fused hands, nightmare anatomy. Fast hand movement near the face also triggers identity resets - when a hand crosses the face, the model treats the next frame as a fresh identity reference. This compounds into drift across the rest of the clip.
✓ Fix: Slow everything down. "Gently raises one hand" works. "Rapidly counts on fingers" does not. Keep hands below the chin. If hands must move, use wide shots where finger detail isn't visible.
Hard Limits | Not Prompt Problems

Things Seedance 2.0
genuinely cannot do.

These aren't things that can be fixed with better prompting. They are architectural limits or policy decisions. Knowing them before you try saves credits and frustration.

✗
Real photorealistic faces - blocked since February 10, 2026
Following the Disney C&D and Hollywood IP controversy, uploads of real, identifiable human faces are blocked by the platform. This is a policy enforcement, not a technical limitation. Workarounds: use illustrated characters, describe appearance in text, or use AI-generated character images as your reference.
✗
Native 4K output
Seedance 2.0 generates at up to 2K (2160 × 1080). True native 4K - not upscaled - requires Kling 3.0. Seedance 2.0's output can be upscaled in post, but this is not the same as native 4K generation.
✗
Clips longer than 15 seconds
15 seconds is the hard maximum per generation. This isn't a free-tier limit - it applies on all plans. For longer content, use scene chaining (Level 3 Playbook), or switch to Kling 3.0 which supports up to 3 minutes via its Extend feature. Drift compounds after 4–5 chained Seedance clips - plan your sequence accordingly.
✗
Frame-level editing
Seedance 2.0 has reference-based editing - you can extend footage, swap elements, and modify scenes using @Video references. What it doesn't have is frame-level tools like Runway's Motion Brush, which lets you paint motion onto specific regions of a frame. If your workflow needs that kind of pinpoint control, Runway Gen-4.5 is the right tool.
✗
Official public REST API (as of April 2026)
ByteDance has not publicly released an official REST API for Seedance 2.0. API access currently runs through third-party providers (PiAPI, Atlas Cloud). BytePlus access exists but uses per-minute billing that makes short clips expensive. This situation is expected to change - check current status before building a production pipeline.

Seedance 2.0 - Cheat Sheet
jesiwicks.com/seedance
INPUT LIMITS
Images: ≤ 9 files
Video: ≤ 3 clips · 15s total
Audio (MP3): ≤ 3 · 15s total
Total files: 12 max
Output: 4–15 seconds · 2K max
Prompt: ≤ 80 words
PROMPT FORMULA
Subject + Action + Scene + Camera + Style
One camera move per prompt. Always.
Replace text with @tag + role when you have a reference.
@ TAG PATTERNS
@Image1 as the character
Replicate camera movement from @Video1
@Image1 as the first frame
Background music refers to @Audio1
Extend @Video1 by 5s
FAST VS STANDARD
Use Fast for all drafts (half cost, 2× speed)
Switch to Standard for final render only
Same capabilities across both variants
NEVER DO THESE
✗ Stack camera movements (pan + zoom + track)
✗ 3+ characters in one scene
✗ Prompts over 80 words
✗ Fast hand gestures / finger counting
✗ Upload files without @ tags in prompt
✗ Use MultiFrames for Seedance 2.0 work
WHEN TO SWITCH MODELS
Need 4K native → Kling 3.0
Maximum photorealism → Veo 3.1
Advanced frame editing → Runway Gen-4.5
High volume / tight budget → Wan 2.6
Clips longer than 15s → Kling 3.0 (up to 3 min)
CHARACTER CONSISTENCY RULES
✓ Use 2 reference images max, not 6
✓ Match lighting temp across all refs
✓ Include character anchor block in every prompt
✓ Slow movement protects facial identity
✓ Avoid fast hand gestures near face
✓ Wide shots hide finger artifacts
