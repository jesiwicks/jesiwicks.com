The Core Concept Mental Model

Seedance 2.0 is reference-first - upload your files, assign each one a role with @ tags, and the model follows your references instead of guessing at them. That's a meaningful difference: 12 files across four input types in a single generation, with an @ reference system that has no direct equivalent in any other model.

The Shift That Changes Everything

Describing guesses.
Directing specifies.

Text prompts leave the decisions to the model - what your character looks like, which camera angle, what "cinematic" means. References take those decisions back. Upload a character and the model holds their face. Upload a clip and it replicates the camera move. Upload audio and it syncs to it. The @ reference system is how you stay in control.

Describing - the model decides

"A woman in a red coat walks through a rainy city street at night, cinematic lighting, dramatic camera move."

The model decides what the woman looks like, which street, which camera move, what "dramatic" means. You get its interpretation - not yours.

Directing - you decide

@Image1 as the character. Scene is the street in @Image2. Replicate camera movement from @Video1. Background music from @Audio1.

You supply the references. The model executes your exact vision - character, scene, camera, and sound - all under your control.

💡 What directing actually does Every @ tag is an instruction the model can't ignore. Tag an image as your character - it locks their face. Tag a clip - it copies the camera move. Tag audio - it syncs to it. Each tag removes one decision from the model and hands it to you.
The @ Tag System

Every file needs
a name and a role.

When you upload files into Omni Reference, each one gets assigned a name automatically. You use that name in your prompt as an @ tag to tell the model exactly what to do with that file. Without the @ tag, the model guesses.

Annotated prompt example each color = one input file
@Image1 as the main character, medium shot walking through a rain-soaked street at night. Replicate the camera movement from @Video1. Background audio: @Audio1 for ambient tone. Slow deliberate pace. Film grain. No motion blur on face.
@Image1: character identity text prompt: what to generate @Video1: camera movement @Audio1: audio reference
Rule 01
Upload first, tag second
Type @ anywhere in your prompt to open the asset picker. The platform assigns names automatically: Image1, Video1, Audio1, in upload order.
Rule 02
Assign a role, not just a tag
The tag alone does nothing. @Image1 as the character, @Video1 for camera movement, @Audio1 for background tone. The role is what the model acts on.
Rule 03
Don't re-describe your references in text
If @Image1 is your character, don't describe the character again in text. The file is your description. Use your prompt words for actions, movement, timing, and atmosphere only.
Rule 04
Use Omni Reference, not MultiFrames
MultiFrames is a Seedance 1.0 carry-over that appears in the UI. For all multimodal work in 2.0 (images, video, audio, or any combination) - use Omni Reference only.
Why Characters Drift | And How to Stop It

The model prioritizes
motion over identity.

This is the most important technical fact about Seedance 2.0 that most guides don't explain. When the model has to choose between keeping a character's face accurate and making the motion feel alive, it picks motion. Understanding this changes how you build your reference set.

Fix 01
Use 2 reference images, not 6
More references give the model more variation to reconcile. Field testing shows cutting from 6 images to 2 consistent, well-lit shots reduces visible drift by roughly 60%. Less is more.
Fix 02
Match lighting across all reference images
Mixing warm indoor and cool outdoor references causes the model to alternate between skin tones and eye colors across frames. Keep lighting temperature consistent across every character reference.
Fix 03
Add a character anchor line to every prompt
Include: Character: [name], [distinctive features]. Keep unchanged. Then: No mirrored features. Seedance respects explicit negative anchors more reliably than most models.
Fix 04
Slow the motion to protect the face
Fast movements near the face (hand gestures, quick yaw turns) trigger identity resets. When a hand crosses the face, the model treats the next frame as a fresh reference. Slower, deliberate motion keeps identity stable.

The Prompt Formula

Five parts.
Under 80 words.

Every effective Seedance 2.0 prompt follows the same structure, regardless of complexity. The formula scales from a basic text-only generation all the way to a multi-file reference shoot. Learn this structure once and every template below will follow logically.

The Universal Prompt Structure
Subject + Action + Scene + Camera + Style
Keep it under 80 words. One camera move per prompt. Be specific about speed: "slowly," "quickly," "gently" - not "dramatically" or "dynamically." If you've uploaded reference files, replace the relevant text description with the @ tag and a role. Don't describe what you've already shown.

Level 3 | Multi-Step Production
Level 3
Chaining clips into a coherent sequence
30–60 minutes · Full workflow · Goal: produce a sequence of clips that hold together visually

Seedance maxes at 15 seconds per generation. For longer content you chain clips. Each generation uses the previous output as a reference. The key is keeping your character block, color grade, and camera logic identical across every generation.

Phase 1 - Before you prompt
Build your reference pack first One character headshot (front-facing, clean lighting), one style reference image, and optionally one 2-3 second video clip for camera language. A disciplined reference pack is the difference between a chain that holds and one that drifts. Trim video refs to only what you need.
Phase 2 - Lock your character block
Character block template - paste into every prompt
Character: [name], [hair color and style], [clothing], [distinctive features]. Keep these features unchanged throughout. No mirrored features.
Phase 3 - Generate and lock Shot 1
Shot 1 - Fast mode draft
@Image1 as the character. [Character block]. [Action for Shot 1 - what happens in this clip]. [Camera direction]. [Lighting and atmosphere]. [Color grade note].
Mode: Fast
Iterations: 2-3 drafts
Change one variable per run
Phase 4 - Chain each subsequent shot
Shot 2+ chaining prompt
Continue the scene from @Video1. @Image1 is the character. [Character block]. [New action for this shot]. Match the color grade and camera height exactly.
Upload before prompting @Video1 = the output from the previous shot · @Image1 = your original character headshot
Phase 5 - Final render
Mode: Standard
Render: one shot at a time
If a shot drifts: redo that shot only
Chain limit: drift compounds after 4-5 clips Each generation introduces small variations. By Shot 5 or 6, even with a consistent reference pack, subtle drift accumulates. For sequences longer than 4-5 clips, rebuild your reference pack from the latest output as your new anchor before continuing.

5 Common Mistakes Hard Limits When to Switch

These are the five mistakes most Seedance 2.0 users make early on. Most of them aren't obvious from the documentation. They come from people burning through credits and documenting what went wrong.

The Five Mistakes

Known failure modes.
Preventable every time.

These are documented from real community experience - not hypothetical edge cases. Every one of them has a specific fix.

01
Forgetting @ tags on uploaded files
You upload a character image and a reference video, write a prompt, and hit generate. The model ignores both files because they were never referenced in the prompt. Without the @ tag, the model doesn't know which file to use for what - it may not use them at all, or mix them up unpredictably.
✓ Fix: Every uploaded file needs an @ tag with a role. Every time. No exceptions. "@Image1 as the character" is the minimum.
02
Too many characters in one scene
Three or more characters in a single generation leads to faces drifting, bodies warping, and someone growing an extra arm. The model's attention is split across too many identity anchors and motion paths simultaneously. It picks motion over identity when attention runs out - and with 3+ characters, it runs out fast.
✓ Fix: Maximum 2 characters per generation. If you need a crowd, make them blurry background elements without identity references. Generate foreground characters separately.
03
Writing a novel instead of a prompt
Prompts over ~100 words cause the model to cherry-pick details at random while ignoring others. The more you add, the less control you have. Creators who are used to Midjourney-style descriptor stacking consistently get worse results in Seedance than those who write tight, structured 40–60 word prompts.
✓ Fix: Subject + Action + Scene + Camera + Style. Under 80 words. If it runs over 80 words, cut it. Move detail into your reference images instead.
04
Stacking camera movements
"Pan left while zooming in as the camera tracks the subject" produces a jittery mess that looks like a broken gimbal. The model can execute one camera movement well. It cannot execute three simultaneously. This is not a prompt wording problem - it is a fundamental model constraint.
✓ Fix: One camera move per prompt. Always. A slow dolly-in. A gentle pan. Or a static lock. If you need multiple moves, they go in separate generations.
05
Fast hand gestures and finger movements
"Rapidly gestures while counting on fingers" → extra fingers, fused hands, nightmare anatomy. Fast hand movement near the face also triggers identity resets - when a hand crosses the face, the model treats the next frame as a fresh identity reference. This compounds into drift across the rest of the clip.
✓ Fix: Slow everything down. "Gently raises one hand" works. "Rapidly counts on fingers" does not. Keep hands below the chin. If hands must move, use wide shots where finger detail isn't visible.
Hard Limits | Not Prompt Problems

Things Seedance 2.0
genuinely cannot do.

These aren't things that can be fixed with better prompting. They are architectural limits or policy decisions. Knowing them before you try saves credits and frustration.

✗
Real photorealistic faces - blocked since February 10, 2026
Following the Disney C&D and Hollywood IP controversy, uploads of real, identifiable human faces are blocked by the platform. This is a policy enforcement, not a technical limitation. Workarounds: use illustrated characters, describe appearance in text, or use AI-generated character images as your reference.
✗
Native 4K output
Seedance 2.0 generates at up to 2K (2160 × 1080). True native 4K - not upscaled - requires Kling 3.0. Seedance 2.0's output can be upscaled in post, but this is not the same as native 4K generation.
✗
Clips longer than 15 seconds
15 seconds is the hard maximum per generation. This isn't a free-tier limit - it applies on all plans. For longer content, use scene chaining (Level 3 Playbook), or switch to Kling 3.0 which supports up to 3 minutes via its Extend feature. Drift compounds after 4–5 chained Seedance clips - plan your sequence accordingly.
✗
Frame-level editing
Seedance 2.0 has reference-based editing - you can extend footage, swap elements, and modify scenes using @Video references. What it doesn't have is frame-level tools like Runway's Motion Brush, which lets you paint motion onto specific regions of a frame. If your workflow needs that kind of pinpoint control, Runway Gen-4.5 is the right tool.
✗
Official public REST API (as of April 2026)
ByteDance has not publicly released an official REST API for Seedance 2.0. API access currently runs through third-party providers (PiAPI, Atlas Cloud). BytePlus access exists but uses per-minute billing that makes short clips expensive. This situation is expected to change - check current status before building a production pipeline.

The @ reference system is how you tell Seedance 2.0 what each uploaded file is for. When you upload files in Omni Reference mode, each gets an automatic name: Image1, Video1, Audio1. You then use those names as @ tags in your prompt to assign each file a role.

For example: @Image1 as the character's face locks that person's appearance. @Video1 for camera movement replicates the camera path from your reference clip. @Audio1 for music tone tells the model to match your audio file's mood.

Without @ tags, uploaded files have no assigned role. The model either ignores them or guesses. The @ tag is the instruction. The file is the reference. Both are required.

Read the full @ reference system guide

Upload up to 3 audio files (MP3, up to 15 seconds each) in Omni Reference mode. Each file gets an automatic name: Audio1, Audio2. Tag them in your prompt to define their role:

@Audio1 for background music tone: the model matches the energy and style of your track.
@Audio1 for dialogue sync: generates lip movement synchronized to spoken words.
@Audio1 for beat sync: times visual cuts and motion to your music's rhythm.

Seedance 2.0 generates audio natively alongside video in a single pass, not as a post-processing step. No separate audio editing required for basic sync.
